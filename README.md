# Реалізовані функції

## 1. find_coins_greedy(amount)

Алгоритм видачі решти, що послідовно використовує найбільші номінали.  
Повертає словник {номінал: кількість}.  
Складність: O(N), де N — кількість номіналів.

### 2. find_min_coins(amount)

Алгоритм динамічного програмування для пошуку мінімальної кількості монет.  
Повертає словник {номінал: кількість}.  
Складність: O(N * S), де S — сума.

Обидва алгоритми повністю виконують вимоги завдання та коректно працюють для будь-якої не від’ємної суми.

## Порівняння алгоритмів

Тестування проводилося на сумах:  
`10, 113, 1000, 10_000, 100_000`.

**Результат:**
Жадібний алгоритм працює стабільно та майже миттєво на будь-яких сумах.  
Динамічне програмування дає оптимальну відповідь, але час виконання та використання пам’яті росте разом із сумою.

**Висновок:**  
Для цього набору монет обидва методи дають оптимальний результат, але при великих сумах ефективнішим є жадібний алгоритм.
